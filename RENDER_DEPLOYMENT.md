# Render.com Deployment Guide

## Step 1: Push Project to GitHub

First, initialize a Git repository (if not already done) and push to GitHub:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/comms-platform.git
git push -u origin main
```

## Step 2: Deploy on Render.com

1. Go to [render.com](https://render.com)
2. Sign up with your GitHub account
3. Click **New +** → **Web Service**
4. Connect your GitHub repo
5. Select the `comms-platform` repository
6. Configure the service:
   - **Name**: `comms-platform`
   - **Environment**: `Python 3`
   - **Region**: Choose closest to you
   - **Branch**: `main`
   - **Build Command**: (Should auto-detect from render.yaml)
   - **Start Command**: `gunicorn comms_platform.wsgi:application --bind 0.0.0.0:$PORT`
   - **Plan**: Free

## Step 3: Configure Environment Variables

In Render dashboard, go to **Environment** tab and add:

```
DEBUG=False
SECRET_KEY=<auto-generated by Render>
ALLOWED_HOSTS=*.onrender.com
RESEND_API_KEY=<your-resend-key>
TWILIO_ACCOUNT_SID=<optional>
TWILIO_AUTH_TOKEN=<optional>
TWILIO_PHONE_NUMBER=<optional>
```

### How to get these keys:

**Resend API Key**: 
- Go to [resend.com](https://resend.com)
- Sign up (free tier available)
- Create API key in dashboard

**Twilio Credentials** (optional):
- Go to [twilio.com](https://twilio.com)
- Sign up for free trial
- Get Account SID, Auth Token, and Phone Number from console

## Step 4: Attach PostgreSQL Database

1. In Render dashboard, click **New +** → **PostgreSQL**
2. **Name**: `comms-platform-db`
3. **Plan**: Free
4. Once created, copy the **Database URL**
5. Go back to your Web Service → **Environment**
6. Add environment variable:
   ```
   DATABASE_URL=<paste-the-database-url>
   ```

## Step 5: Deploy

1. Click **Deploy** button
2. Wait for build to complete (2-3 minutes)
3. Once deployed, Render gives you a URL like: `https://comms-platform.onrender.com`

## Step 6: Create a Superuser (Admin Account)

Once deployed, you need to create a superuser for the Django admin:

```bash
# Option 1: Use Render Shell
# In Render dashboard → Web Service → "Shell" tab, run:
python manage.py createsuperuser

# Option 2: Use SSH into the server (for manual setup if needed)
```

## Step 7: Access Your App

- **App**: `https://comms-platform.onrender.com`
- **Admin Panel**: `https://comms-platform.onrender.com/admin`
- **Dashboard**: `https://comms-platform.onrender.com/dashboard` (after login)

## Troubleshooting

### Build Fails
- Check **Build Logs** in Render dashboard
- Ensure `requirements.txt` has all dependencies
- Verify `render.yaml` is correctly formatted

### App crashes
- Check **Logs** tab in Render dashboard  
- Common issue: Missing environment variables
- Add `DEBUG=True` temporarily to see error details

### Database Issues
- Ensure `DATABASE_URL` is set in environment
- Render automatically runs migrations on deploy
- To manually migrate: Use **Shell** tab and run `python manage.py migrate`

## Cost

- **Web Service**: Free tier (with limitations)
  - Will sleep after 15 min of inactivity
  - Limited to 0.5GB memory
  - 100GB/month bandwidth
  
- **PostgreSQL**: Free tier
  - 256MB storage
  - Automatic backups

For continuous uptime, upgrade to **Starter** plan ($7/month) or use another platform.
